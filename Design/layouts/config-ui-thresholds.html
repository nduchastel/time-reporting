Configuration System Design:

  • Hierarchical Rules: Global → Seasonal → Worksite → Worker (most specific wins)
  • Optional Overrides: Only global defaults are required; everything else is optional
  • Flexible & Extensible: Easy to add/remove rules, configure thresholds per entity
  • Review Card Generation: Each violated rule creates a Review Card for manager approval
  • Clear Hierarchy: Visual color-coding shows rule priority (blue=global, green=seasonal, teal=worksite, worker-specific colors)
