Worker & Worksite Management Features:

  • Edit Anytime: All worker/worksite information and rules can be edited after initial setup
  • Custom Rules Per Entity: Override global thresholds for specific workers or worksites
  • Temporal Disable: Disable workers or worksites for specific date ranges (auto re-enable) or indefinitely
  • Scheduled Time Off: Pre-configure worker vacation days - system won't flag missing time entries
  • Operating Days/Hours: Set per-worksite rules for when the site is open
  • Worksite Aliases: Map multiple ways workers might say a site name ("Simons", "Simons site", "Main Street property")
  • All Changes Logged: Every edit, disable, enable, or delete generates a log event tagged to the affected entity
  • Delete vs Disable: Delete removes entity permanently (with confirmation); Disable keeps history intact

Log Events Generated:

• Worker edited (name/contact changed):
  Tags: worker:bob-martinez, config:basic-info

• Worker rules updated (max hours changed):
  Tags: worker:bob-martinez, config:per-worker-rules, rule:max-hours-week

• Worker disabled (vacation May 24-30):
  Tags: worker:bob-martinez, action:disable
  Details: {reason: "Vacation", until: "2026-05-30"}

• Worksite closed (temporarily):
  Tags: worksite:johnston-site, action:disable
  Details: {reason: "Renovations", until: "2026-06-01"}
