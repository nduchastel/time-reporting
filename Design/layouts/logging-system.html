Logging System Design Principles:

  • Many-to-Many Tags: Each log event can be tagged with unlimited entities (workers, worksites, cards, rules, etc.)
  • Extensible Tag Types: New tag types can be added without schema changes (equipment, projects, clients, invoices)
  • Queryable: Filter logs by any combination of tags, date ranges, actions, or actors
  • Complete Audit Trail: Before/after state capture for all changes
  • Compliance-Ready: Export logs for labor audits, disputes, or payroll verification
  • Performance: Tag index enables fast lookups: "Show all events for worker X" or "Show all config changes"

Example Queries:

• Get all events for Bob Martinez:
  tags contains "worker:bob-martinez"

• Get all overtime overrides:
  action = "override_validation_rule" AND tags contains "rule:max-hours-day"

• Get all changes to Simons Property config:
  tags contains "worksite:simons-property" AND tags contains "config"

• Get all events involving multiple workers:
  tags contains "worker:*" (count > 1)

• Get all deletions in last 30 days:
  action = "delete" AND timestamp > "30 days ago"
